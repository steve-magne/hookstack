# Channels — tactiques par canal

Trois canaux retenus : **X**, **Reddit/HN**, **LinkedIn**. Chacun a ses codes. Poster le même texte partout = échec garanti. `/growth-post` adapte au canal selon ce fichier.

---

## X / Twitter — cœur de la communauté Claude Code

**Pourquoi** : c'est là que vit la communauté IA agentique / Claude Code. Le canal #1 pour un dev-tool early.

**Format qui marche**
- **Thread** (pas un tweet isolé). 1ère ligne = l'accroche (cf. banque d'angles). Si la 1ère ligne ne fait pas s'arrêter le scroll, le reste est invisible.
- **Média obligatoire** : GIF de démo (≤ 15s) ou capture de code. Les tweets avec média font 2–3× l'engagement.
- **1 seul CTA**, dans le dernier tweet du thread (pas le premier — l'algo pénalise les liens sortants en tête).
- Pas de hashtags spam (1 max, ou zéro).

**Cadence & timing** : 3–4×/sem. Mardi–jeudi, 9–11h ET (pic dev US). Engager les réponses dans l'heure.

**Tactiques de levier**
- Répondre intelligemment aux gros comptes Claude Code / Anthropic (value-add, pas « check my project »).
- Quote-tweet les annonces Anthropic avec un angle hook concret.
- Tagger avec parcimonie quand c'est pertinent et mérité.

## Reddit & Hacker News — le canal des spikes

**Pourquoi** : un hit front-page = 200–1 500 stars en 24h. Haut risque, haute récompense. **Le canal le plus susceptible de faire basculer le repo en trending.**

**Règles absolues (les enfreindre = ban / downvote to death)**
1. **Value-first, toujours.** Jamais « regardez mon projet ». Toujours « voici un truc utile » où le projet est le moyen, pas le message.
2. **Disclose que tu es l'auteur.** « I built this » — l'honnêteté est récompensée, le faux astroturfing est puni.
3. **Sois présent les 2 premières heures.** Réponds à *chaque* commentaire. Le ranking HN/Reddit dépend de la vélocité initiale.
4. **Un subreddit à la fois.** Pas de cross-post simultané (filtre anti-spam).

**Hacker News — Show HN**
- Titre : `Show HN: Hookstack – a community catalogue for Claude Code hooks`
- Lien : le repo OU un writeup (pas une landing marketing).
- 1er commentaire (toi) : le contexte en 3–4 lignes — quel problème, pourquoi tu l'as construit, ce qui est gratuit/open-source.
- Timing : mar–jeu, 8–10h ET. Ne pas demander d'upvotes (bannissable).

**Reddit**
- r/ClaudeAI : le plus réceptif, partage genuine OK.
- r/SideProject, r/programming : substance réelle requise, ton plus modeste.
- Lis les règles de chaque sub (certains interdisent l'auto-promo → poste de la valeur sans lien, lien en commentaire si demandé).

## LinkedIn — crédibilité & persona « AI Champion »

**Pourquoi** : moins viral, mais touche la **persona 3** (Architecte Platform / responsable adoption Copilot/Claude en entreprise). Bon pour la crédibilité et le réseau pro.

**Format qui marche**
- **Native, story-driven.** Une galère vécue → ce que tu as construit → la leçon. Personnel, à la 1ère personne.
- **Pas de lien dans le corps** : LinkedIn enterre les posts avec liens sortants. → lien en **1er commentaire**.
- Angle « équipe/organisation » : « comment standardiser les hooks Claude Code dans une équipe » résonne mieux ici que sur X.
- Longueur moyenne (800–1 300 caractères), retours à la ligne aérés.

**Cadence** : 1–2×/sem. Mardi–jeudi matin.

## Newsletters & dev.to (outreach, pas posting)

Non sélectionné comme canal primaire, mais **fort levier passif** via `/growth-outreach` :
- Pitch court aux curateurs : **TLDR**, **Console.dev**, **Bytes**, **Changelog News**. Email 4 lignes + 1 GIF + 1 lien. Gratuit. Un feature TLDR = pic massif.
- dev.to / Hashnode : republier le cornerstone (canonical link vers le blog). SEO long-terme.

## Matrice de réutilisation (1 idée → N formats)

| Sujet source | X | LinkedIn | Reddit/HN | Site/README |
|---|---|---|---|---|
| Un hook concret | Thread + GIF | Story « pourquoi ce hook » | Commentaire utile dans un thread pertinent | Entrée catalogue |
| Le cornerstone (5 hooks) | Thread résumé | Post leçon | Show HN / r/ClaudeAI | Section README |
| Un chiffre (62 hooks, X installs) | Tweet build-in-public | Post milestone | — | Badge / social proof |

> Ne jamais créer du contenu *ex nihilo* à chaque fois. Pars d'un actif (un hook, le cornerstone, un chiffre) et **atomise**-le sur les canaux. C'est ce que fait `/growth-post`.
