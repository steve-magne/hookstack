# 05 — UX d'implémentation des hooks sélectionnés

## Décision retenue : CLI `npx hookstack-cli@latest`

**Statut : implémenté et en production.**

Le deliverable de Hookstack est une commande `npx` que l'utilisateur colle dans son terminal :

```bash
npx hookstack-cli@latest install --hooks=<slug1>,<slug2>,...
```

Cette commande :
1. Télécharge les scripts `.mjs` des hooks sélectionnés
2. Les place dans `.claude/hooks/` du projet courant
3. Patche `.claude/settings.json` en mergant intelligemment (regroupe par event + matcher, no écrasement)
4. Affiche un résumé de ce qui a été fait

## Pourquoi ce choix

| Critère | Bénéfice |
|---|---|
| Zéro friction | `npx` ne nécessite pas d'installation préalable |
| OS-agnostique | Fonctionne partout où `node` est dans le PATH |
| Merge intelligent | Préserve le `settings.json` existant |
| Maintenable | Le CLI est versionné dans `packages/cli/` |
| Identité produit | L'URL Hookstack + la commande = loop viral naturel |

## UX dans le site

`HookConfigurator.tsx` (ligne 21) construit `pluginCmd` en temps réel à partir des slugs sélectionnés. La bannière sticky (effet ⑧ dans DESIGN.md) pulse à chaque sélection/désélection pour attirer l'attention sur la commande mise à jour.

Trois boutons de copie coexistent :
- **Copier la commande** (principal) — `npx hookstack-cli@latest install --hooks=...`
- **Copier le settings.json** (secondaire) — pour inspection ou merge manuel
- **Copier les scripts** — les `.mjs` individuels

## Fichiers clés

- `HookConfigurator.tsx` — génère et affiche la commande
- `src/lib/mergeConfig.ts` — logique de fusion des configs
- `src/lib/hookExports.ts` — exports des hooks pour le CLI
- `packages/cli/` — code source du CLI `hookstack-cli`

## Options analysées (archivées)

Ces options ont été étudiées avant la décision finale :

| Solution | Setup requis | UX | Merge intelligent | Coût maintenabilité |
|---|---|---|---|---|
| **NPX CLI** ✅ | Zéro | Terminal | Oui | Moyen (npm publish) |
| Script node généré | Zéro | Coller dans terminal | Partiel | Très faible |
| Prompt Claude Code | Zéro | Coller dans Claude | Oui (IA) | Très faible |
| Skill `implement-hooks` | Une fois | `/implement-hooks` | Oui (IA) | Moyen |
| Plugin MCP | Lourd | Automatique | Oui | Très élevé |

Le "script node généré" était la recommandation initiale — abandonné en faveur du CLI pour des raisons d'identité produit (URL mémorisable, versioning npm, loop viral).
