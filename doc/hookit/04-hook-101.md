## Taxonomie des hooks (scope POC v1)

Le POC se concentre sur **Claude Code** et **GitHub Copilot VS Code**, les deux providers les mieux maîtrisés.

### Catégories principales

| Catégorie | Exemples de hooks | Cas d'usage |
|---|---|---|
| **Sécurité** | `PreToolUse: Bash` — détection de secrets | Bloquer les commandes destructives, valider les sorties |
| **Contexte** | `UserPromptSubmit` — injection de contexte repo | Ajouter les conventions du projet, le stack tech |
| **Validation** | `PostToolUse: Write` — lint auto après écriture | Déclencher des vérifications qualité |
| **Notification** | `SessionStart`, `SessionStop` — alertes | Logger les sessions, notifier sur Slack |
| **Workflow** | `PreToolUse: WebFetch` — proxy de requêtes | Contrôler les accès réseau des agents |
| **Documentation** | `Stop` — génération de résumé | Créer un changelog automatique en fin de session |

### Structure d'un hook dans le registre

### Hooks universels vs. tech-spécifiques

Un hook **sans `stack`** est universel : il s'affiche quel que soit le projet. Un hook **avec `stack`** (`"typescript"`, `"node"`, `"python"`) n'est affiché que si l'utilisateur a sélectionné cette stack dans le catalogue — c'est le mécanisme opt-out du filtre "Your stack".

Règle : annoter `stack` uniquement quand le `code_snippet` le confirme (filtre d'extension, outil non universel). Ne jamais se fier aux `tags` seuls — les agents d'analyse peuvent taguer `pytest` sur un script qui ne lance que Vitest.

```json
{
  "id": "pre-bash-secret-detection",
  "name": "Détection de secrets avant exécution Bash",
  "category": "security",
  "provider": ["claude-code", "copilot-vscode"],
  "hook_type": "PreToolUse",
  "trigger": "Bash",
  "description": "Intercepte toute commande shell et analyse le prompt pour détecter des clés API, tokens ou mots de passe avant exécution.",
  "use_cases": ["CI/CD sécurisé", "Protection de credentials", "Audit DevSecOps"],
  "implementation": {
    "type": "settings_json",
    "config": {},
    "code_snippet": "..."
  },
  "community_examples": [
    { "repo": "github.com/...", "issue_url": "...", "added_by": "claude-code-analysis" }
  ],
  "tags": ["security", "bash", "secrets", "devsecops"],
  "stack": ["typescript"]
}
```
